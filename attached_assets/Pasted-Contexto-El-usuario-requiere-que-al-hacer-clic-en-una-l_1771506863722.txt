Tarea 1: Fix Definitivo del Error 500 (Backend)
Para evitar excepciones de referencia nula en la proyección LINQ o problemas de reflexión con Native AOT, vamos a obtener la entidad de la base de datos primero y mapearla a DTO en memoria.
Archivo: StatsController.cs o el archivo de Minimal APIs correspondiente. Acción: Reemplaza el endpoint GET /api/stats/labors/{id} con esta versión a prueba de fallos:
app.MapGet("/api/stats/labors/{id}", async (Guid id, AppDbContext db) =>
{
    // 1. Traer la entidad a memoria con sus relaciones (sin proyecciones LINQ complejas)
    var labor = await db.Labors
        .AsNoTracking()
        .Include(l => l.Lot)
        // .Include(l => l.Responsible) // Descomentar si existe la relación de Responsable
        .FirstOrDefaultAsync(l => l.Id == id);

    if (labor == null) return Results.NotFound();

    // 2. Mapear a DTO en memoria (C# puro, evita fallos de traducción SQL)
    var dto = new LaborDetailDto
    {
        LaborId = labor.Id,
        Type = labor.Type,
        Status = labor.Status,
        PlannedDate = labor.PlannedDate,
        LotName = labor.Lot?.Name ?? "Sin Lote Asignado",
        LotId = labor.LotId,
        // Ajustar según el modelo exacto de tu BD:
        ResponsibleName = "Sin Asignar", // Reemplazar con labor.Responsible?.Name si existe
        Machinery = labor.Machinery ?? "N/A",
        Inputs = new List<string>() // Rellenar si existe tabla de insumos
    };

    return Results.Ok(dto);
});
Nota para el agente: Asegúrate de que LaborDetailDto esté registrado en AppJsonSerializerContext para la compilación AOT.

Tarea 3: Integrar el Modal en el Calendario (WorkPlanner.razor)
Objetivo: Al hacer clic en la "pastilla" (chip) de una labor en el calendario, se debe hacer el fetch del detalle y mostrar el modal.
Archivo: Client/Pages/WorkPlanner.razor Acciones:
1. Añadir el estado para almacenar el detalle cargado: private LaborDetailDto? SelectedLaborDetail;
2. Crear la función para abrir el detalle:
private async Task OpenLaborDetail(Guid laborId)
{
    try
    {
        // Llamada al endpoint corregido
        SelectedLaborDetail = await Http.GetFromJsonAsync<LaborDetailDto>(
            $"api/stats/labors/{laborId}", 
            AppJsonSerializerContext.Default.LaborDetailDto
        );
    }
    catch (Exception ex)
    {
        Console.WriteLine($"Error cargando detalle: {ex.Message}");
    }
}
3. En el bloque HTML del calendario, actualizar el evento @onclick de la labor para que llame a esta nueva función:
