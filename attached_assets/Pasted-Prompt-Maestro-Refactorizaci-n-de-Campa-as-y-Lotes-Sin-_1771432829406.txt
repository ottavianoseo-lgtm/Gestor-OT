Prompt Maestro: Refactorización de Campañas y Lotes (Sin Módulos Nuevos)
Objetivo: Integrar la gestión de superficies dinámicas y rotaciones dentro de los módulos existentes Campaigns y Plots en .NET 10 y Blazor, modificando el modelo de datos para soportar la asignación específica por ciclo.
1. Base de Datos (PostgreSQL)
No crear tablas de "Planificación" sueltas. Usaremos una tabla relacional con carga útil (Payload) que vincule Lotes y Campañas.
-- Tabla intermedia con "payload" (datos de negocio)
CREATE TABLE "CampaignPlots" (
    "Id" UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    "CampaignId" UUID NOT NULL REFERENCES "Campaigns"("Id"),
    "PlotId" UUID NOT NULL REFERENCES "Plots"("Id"),
    "CropId" UUID NULL REFERENCES "Crops"("Id"), -- El cultivo planificado (La Rotación)
    
    -- LA CLAVE: Superficie que vale para ESTA campaña
    "ProductiveSurfaceHa" DECIMAL(10,2) NOT NULL, 
    
    -- Fechas estimadas agronómicas (independientes de la fiscalidad de la campaña)
    "EstimatedStartDate" DATE,
    "EstimatedEndDate" DATE,

    CONSTRAINT "UK_Campaign_Plot" UNIQUE ("CampaignId", "PlotId")
);
2. Backend (.NET 10 - Minimal API)
Modificación en CampaignsController
Agregar endpoints para gestionar la configuración interna de la campaña.
• POST /api/campaigns/{id}/plots: Recibe una lista de Lotes con su ProductiveSurfaceHa y CropId.
    ◦ Lógica: Al guardar, usar ExecuteUpdate (EF Core 10) si ya existen registros para actualizar masivamente superficies sin traer entidades a memoria.
• GET /api/campaigns/{id}/fields/{fieldId}/surface: Calcula la superficie del campo sumando CampaignPlots.ProductiveSurfaceHa (No usar Plots.Surface).
Modificación en PlotsController
Agregar endpoint para consultar el historial (Rotación).
• GET /api/plots/{id}/history: Devuelve una lista ordenada por fecha de CampaignPlots (incluyendo nombre de Campaña y Cultivo). Esto construye la vista de rotaciones.
3. Frontend (Blazor - Módulos Existentes)
A. Módulo Configuración de Campaña (CampaignConfig.razor)
• Grid de Asignación: Dentro del detalle de una campaña, agregar una sección "Lotes Asignados".
• Edición en Línea: Usar un componente QuickGrid donde el usuario pueda:
    1. Ver la superficie real (Catastral) del lote (Solo lectura).
    2. Editar la Superficie Productiva (Input editable) para ese ciclo.
    3. Seleccionar el Cultivo (Dropdown).
B. Módulo de Lotes (PlotDetail.razor)
• Pestaña "Rotaciones": No crear un menú nuevo. Dentro del detalle del lote, agregar un tab que consuma GET /api/plots/{id}/history.
• Visualización: Mostrar una línea de tiempo simple: "Campaña 2024: Soja (50ha) -> Campaña 2025: Maíz (50ha)".

--------------------------------------------------------------------------------
Nota de Arquitectura: Para las labores que cruzan campañas: Asegúrate de que el modelo Labor (Labor) ahora tenga una FK nullable a CampaignPlotId. Esto permitirá que una labor de "Barbecho" hecha en diciembre se asigne al cultivo de "Maíz" de la campaña siguiente, manteniendo la integridad de costos.
