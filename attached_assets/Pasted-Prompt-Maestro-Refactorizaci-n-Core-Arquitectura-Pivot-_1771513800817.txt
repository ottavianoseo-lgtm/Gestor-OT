En el DbContext, configura una restricción única para evitar que un lote se agregue dos veces a la misma campaña:
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<CampaignLot>()
        .HasIndex(cl => new { cl.CampaignId, cl.LotId })
        .IsUnique();
}
3. Lógica de Negocio (Backend API)
Implementa los dos flujos operativos clave solicitados:
A. Endpoint: Importar Lotes de la Campaña Anterior
POST /api/campaigns/{newCampaignId}/import-lots/{sourceCampaignId}
• Regla: Copiar los registros CampaignLot de la campaña origen, manteniendo por defecto la misma ProductiveSurfaceHa que tenían (o la catastral si es la primera vez).
app.MapPost("/api/campaigns/{newId}/import-lots/{sourceId}", async (Guid newId, Guid sourceId, AppDbContext db) => 
{
    var existingLots = await db.CampaignLots
        .AsNoTracking()
        .Where(cl => cl.CampaignId == sourceId)
        .ToListAsync();

    var newCampaignLots = existingLots.Select(cl => new CampaignLot
    {
        Id = Guid.NewGuid(),
        CampaignId = newId,
        LotId = cl.LotId,
        // Hereda la superficie productiva del año pasado por defecto
        ProductiveSurfaceHa = cl.ProductiveSurfaceHa 
    });

    await db.CampaignLots.AddRangeAsync(newCampaignLots);
    await db.SaveChangesAsync();
    return Results.Ok();
});
B. Cálculo de Dosis con Precisión Milimétrica (El Impacto)
Cuando se cree una nueva Labor, los cálculos de insumos deben multiplicar la Dosis por la ProductiveSurfaceHa.
app.MapPost("/api/labors", async (LaborCreateDto dto, AppDbContext db) =>
{
    // Obtener el Lote-Campaña para saber el área real a trabajar
    var campaignLot = await db.CampaignLots.FindAsync(dto.CampaignLotId);
    if (campaignLot == null) return Results.BadRequest("Contexto de campaña no válido.");

    var labor = new Labor 
    {
        CampaignLotId = dto.CampaignLotId,
        Type = dto.Type,
        // El costo/dosis se calcula sobre los 90ha, no sobre los 100ha del Lote físico
        CalculatedInputTotal = dto.InputRatePerHa * campaignLot.ProductiveSurfaceHa 
    };

    db.Labors.Add(labor);
    await db.SaveChangesAsync();
    return Results.Ok(labor);
});
4. Frontend (Blazor)
En la vista de "Planificación de Campaña", utiliza un QuickGrid que reciba la lista de CampaignLots de la campaña activa.
• Columna 1: Nombre del Lote (Lot.Name).
• Columna 2: Superficie Catastral (Solo Lectura, Lot.CadastralSurfaceHa).
• Columna 3: Superficie Productiva (Input editable asociado a ProductiveSurfaceHa). Al cambiar este valor, el usuario define que este año "la laguna ocupa 10 ha", actualizando el valor a 90 ha.
